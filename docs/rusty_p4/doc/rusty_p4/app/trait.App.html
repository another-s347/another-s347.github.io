App is the core concept to build a controller, which is for custom logic and function. It can have dependency and option (also called configuration). Using dependency app can get instance of other apps when initializing. So a app is required to be clone + Send + Sync, as it might be held by multiple apps and accessed from different threads.

type Container: App + Clone[src]

To support special function (like optional dependency), some generic container type (like Arc or Option) can also be defined as app. If you create a container type (Option<T>), use this field to specify the target type (T). For regular app, this field should be Self.

type Dependency: Dependencies[src]

Use this field to specify your dependency. It should be a variadic tuple, but variadic tuple is not supported currently. So now, rusty-p4 use crate tuple_list to define dependencies. Use () for no dependency.

fn from_inner(app: Option<Self::Container>) -> Option<Self>[src]

For container type, use this method to convert from target type to container type (T -> Option<T>). For regular type, simplely return will do. This method is called by rusty-p4 internally.
